--- title: Adding a formatter layout: piecrust_code --- To add a new text formatter to **PieCrust**, add a new file in the `_piecrust/formatters` directory. The file should be named `MyAwesomeFormatter.class.php`, like the other formatters. The main class inside that file should be named the same (_e.g._ `MyAwesomeFormatter`) and should implement the `IFormatter` interface: {% geshi 'php' %} class MyAwesomeFormatter implements IFormatter { protected $pieCrust; public function initialize(PieCrust $pieCrust) { // Initialize the formatter. $this->pieCrust = $pieCrust; } public function getPriority() { // Return the priority (high, default or low). return IFormatter::PRIORITY_DEFAULT; } public function supportsFormat($format, $isUnformatted) { // Return whether you want to support the given format return $isUnformatted && ($format == "awesome"); } public function format($text) { // Here, actually for the formatting and all the // expensive stuff. require_once 'my-external-lib.php'; return ExternalLib::doStuff($text); } } {% endgeshi %} Here are a few important things to know: * Because all formatters are initialized, only do cheap stuff in the `initialize` method. Generally speaking, you should only query a few configuration settings from the application object or, better, just store a reference to it for later use. * Formatters are asked in order of priority whether they want to support a format. There are only 3 priorities so far (high, default and low). The **SmartyPants** and _pass-through_ formatters have low priorities. All the other ones have default priority. * The `supportsFormat` function should obviously return `true` if the text should be processed by this formatter. Usually, you want to only process unformatted text (**SmartPants** is an exception, since it adds more formatting on top of other formattings). * The `format` function should do all the expensive stuff -- not only format the text, of course, but also include and load any needed external libraries. For example, you should avoid including external PHP scripts until you know this formatter is going to be used. To test your formatter, just create a page that specifies `awesome` (or whatever string you're matching in `supportsFormat`) as its format.
